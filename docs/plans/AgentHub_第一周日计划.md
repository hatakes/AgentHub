# AgentHub 第一周日计划

> 周期：2026-06-15 至 2026-06-21  
> 目标：用一周时间建立对 AgentHub 当前代码和文档的基本掌控，能运行核心样板、说清执行链路，并产出下一周可落地任务清单。

## 使用方式

```text
每天建议投入 1-2 小时。
每一天只追求一个明确结果。
如果当天时间不够，优先完成“验收标准”，补充阅读可以顺延。
AI 可以用来加速定位、解释和生成草稿，但最终链路必须能自己复述。
```

## 本周总目标

```text
1. 跑通当前项目测试和附件业务样板。
2. 看懂 /agent/chat 到 AgentRuntime 再到 Tool 执行的主链路。
3. 看懂 agent-document-processing 的业务样板边界。
4. 整理出下一周可以实现的可观测性或 RAG 最小任务。
```

## AI 使用边界

```text
适合依靠 AI：
定位文件和入口。
解释陌生代码。
总结长文档。
生成流程图草稿。
整理测试失败日志。
生成 curl 示例。
把口头理解整理成 Markdown。

必须自己完成：
运行命令。
确认测试是否真的通过。
打开关键类读核心方法。
用自己的话复述主链路。
判断任务优先级。
决定下一步做什么。
```

## 第一周结束后的判断标准

```text
通过：
能运行测试或明确测试失败原因。
能说清 AgentRuntime 主链路。
能说清 agent-document-processing 的业务边界。
能列出下一周可执行任务。

不通过：
只看了文档但没有跑命令。
只知道模块名，不知道请求如何进入 Runtime。
下一周任务仍是“学习 RAG”“研究 Workflow”这类不可验收目标。
```

## 第二周候选方向

优先级建议：

```text
P0：Agent Run / Step / ToolCallLog 持久化设计和最小实现
P1：统一错误码和 traceId
P2：RAG 最小闭环设计
P3：附件助手二期能力拆分
```

第二周不建议同时做多个大方向。更推荐先选 P0，把执行过程可观测性补起来。
